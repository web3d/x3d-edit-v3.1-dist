New features

 

Since Xeena 1.2EA

  • XML Schema support is deprecated
  • One distribution for all platforms
  • New license bundled with the distribution

 

Since Xeena 1.1

 
Error Handling
Xeena prompts not only that the node is incomplete but also where is the child that is causing the error.

Command Line Usage

  • You can specify a dtd config file via -profile. Xeena will use it instead of the config file located in USR_HOME or dtd directory.
  • You can run Xeena from any directory. In Windows XEENA_HOME is set during installation. You need to set it by hand (for UNIX see xeena.sh, for Windows see xeena.bat).

Xeena's Extensibility

  • New methods were added to XMLEditorContext to enable to print messages in the Message Area.

Additional Features

  • The default dtd configuration file presents the attributes in the order they appeared in the dtd.
  • Menu/Tool bar actions to print (tree and XML source) and search were added.
  • Menu actions that enable to view the DTD.
  • Apply button was removed from attributes edit window.
  • Keyboard Shortcuts.
  • If no start file was specified in the cmd-line (via -xml flag) Xeena starts with a new document.
  • Elements that can be inserted as children are displayed in bold in the side-bar
    palette (other are in plain text).

DTD Configuration updates

  • In the default dtd configuration file (created by xeena) the 'attribute' elements are in the order they appeared in the dtd.

Since Xeena 1.0.3b

Document Look

  • The whole xml document is visible starting from the xml PI.
  • The root element can be deleted/replaced.
  • The DOCTYPE element appears in the tree. The system-id and public-id are editable.
  • PI's and comments can now be added before/after the root.


Command-Line Usage

  • The -root cmd line option is not mandatory - if not present any element may be root.
  • URLs can be used via the -dtd and -xml flags.

Xeena's Extensibility

  • Activation of tools from Xeena is supported.


Sample built in tools are an xsl-processor and a BML-player.

  • Activation of document importers is supported.


Built in importers are File and URL. A sample built in importer is a BML-importer.

  • Activation of exporters is supported. A sample built in exporter is a BML-exporter.


You can specify what tools,importers and exporters to use via the dtd-configuration file.

Additional Features

  • URL support:
    • Can be used In the command line via -dtd and -xml.
    • Can be opened/imported via the Menu.
    • Can be provided in the config file for  document templates.
    • Can be provided in the config file for icons.
  • Attributes are written to files according to the order specified by the dtd configuration file.
  • Menu/Tool bar actions that enable to collapse/expand nodes were added.
  • Item editors can now be configured to display either in a separate dialog, or in the Edit area, using the item-editor attribute of the Xeena-profile element.


DTD Configuration updates

  • Users can set default values for the system-id and public-id of new DOCTYPE elements (via the dtd-configuration file).
  • Users can set the default value for the encoding of new documents (via the dtd-configuration file).
  • New elements were added to support specification of defaults, importers, exporters and tools. Old dtd configuration files can be used on Xeena 1.1 as all new elements are optional.
  • In order to support DOCTYPE's in palettes the value XML_DOCUMENT_TYPE is now also allowed (in addition to 'old' XML_PI , XML_TEXT etc.) as the value of the attribute 'name' in the 'element' element.
  • Changes in the default dtd configuration file (created by xeena):
    • Contains empty  'importers',  'exporters' elements.
    • Contains  a 'tools' element with an XSL processor tool as a child.
    • Contains a  'defaults' element.
    • The 'element' elements and the  'attribute' elements are sorted by their name.
    • The  'group' element (child of  'sidebarpalette' element) contains a sorted list of all the elements defined in the dtd.

Since Xeena 1.0.2

  • Xeena can be customized via XML files using Xeena itself as the editor. You can customize more aspects of Xeena's behavior by invoking Xeena to create either a DTD-specific customization file or a Xeena-general customization file. More customizable parts will be added in future releases. See the help file for more details.
  • It is now possible to save incomplete files, which can later be loaded and completed.
  • Attributes are now shown and edited in the main Xeena window, rather than in a separate frame.
  • Nodes are constantly validated during editing and underlined in red in case there are invalid. This new feature gives the user immediate feedback on the validity of the edited document.

Since Xeena 1.0

  • Validation of elements - elements can now be independently validated using the "Validate" menu item/toolbar button/popup menu item.
  • Document encoding is now supported - via the "Encoding" menu item.

Underlying package upgrades


Since Xeena 1.0.3b

  • Xeena now uses the XML parser - xerces 1.0.3
  • Xeena now uses the XSL processor - xalan 1.0.1

Since Xeena 1.0.3b

  • Xeena now uses XML4J 2.0.15.

Since Xeena 1.0.2

  • A workaround for a bug in JDK 1.2.2 has been added, so Xeena can now  run with this version of the JDK.

Since Xeena 1.0.1

  • Xeena can now run on Java 2, as well as JDK 1.1. swingall.jar from Swing version 1.1.1 is still included in the release for the benefit of JDK 1.1 users.
  • Xeena now uses XML4J 1.1.16.

Bug fixes

Since 1.2EA

  • spaces preserved in CDATA elements

Since Xeena 1.0.1

  • Default value of attributes not automatically inserted.
  • Text/CDATA cannot be inserted.


Since Xeena 1.0

  • Xeena hangs on Unix when opening the attribute dialog.
  • Out of Memory error on Windows
  • DTD file name changed to lower case.

Changes

  • PopupMenu does not pop up on left mouse click (only on right mouse click).
  • Close option on frame handle of documents is enabled.
  • Automatic selection of the first of enumerated attribute values is canceled.
  • Using Swing's default color scheme.
